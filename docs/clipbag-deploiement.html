Railway — Étape 1
Créer la base de données
On commence par la PostgreSQL. Railway la crée en 30 secondes et te donne une URL de connexion directement.
🗄️
railway.app
Créer la base PostgreSQL
  • 1
    Va sur railway.app → New Project → Deploy PostgreSQL.
  • 2
    Railway crée une instance PostgreSQL. Clique dessus → onglet Variables. Tu verras une variable DATABASE_URL générée automatiquement.
  • 3
    Copie cette DATABASE_URL — elle ressemble à :
    postgresql://postgres:motdepasse@containers-us-xxx.railway.app:5432/railway
  • 4
    Garde-la précieusement, tu en auras besoin dans les variables d'environnement de ton NestJS.
Connecter depuis ton NestJS — app.module.ts
// app.module.ts — TypeOrmModule TypeOrmModule.forRoot({ type: 'postgres', url: process.env.DATABASE_URL, ssl: { rejectUnauthorized: false }, // obligatoire Railway autoLoadEntities: true, synchronize: process.env.NODE_ENV !== 'production', })
⚠️
Ne mets jamais synchronize: true en production. Ça peut effacer des colonnes. Utilise les migrations TypeORM à la place une fois en prod.
Railway — Étape 2
Déployer le backend NestJS
Railway détecte automatiquement NestJS depuis ton GitHub et build le projet. Il faut juste lui dire comment le lancer.
⚙️
Railway
Déployer depuis GitHub
  • 1
    Dans ton projet Railway → New Service → GitHub Repo. Connecte ton compte GitHub et sélectionne first-ecommerce-site.
  • 2
    Railway va demander le Root Directory. Mets backend (c'est le dossier NestJS dans ton repo).
  • 3
    Dans Settings → Build Command : npm run build
  • 4
    Dans Settings → Start Command : node dist/main.js
  • 5
    Dans Settings → Port : 3000 (ou le port de ton NestJS — vérifie dans main.ts).
Vérifier ton main.ts NestJS
// backend/src/main.ts async function bootstrap() { const app = await NestFactory.create(AppModule); app.enableCors({ origin: [ 'https://clipbag.fr', 'https://www.clipbag.fr', 'http://localhost:5173', // dev local ], }); const port = process.env.PORT || 3000; // Railway injecte PORT await app.listen(port); } bootstrap();
💡
Railway injecte automatiquement la variable PORT. Assure-toi que ton NestJS l'utilise sinon le health check échoue et le déploiement boucle.

Vercel — Étape 1
Déployer le frontend Vue.js
Tu connais déjà Vercel — la seule différence ici c'est de pointer sur le dossier frontend de ton monorepo.
▲
vercel.com
Import du projet
  • 1
    Va sur vercel.com → Add New Project → Sélectionne ton repo first-ecommerce-site.
  • 2
    Vercel va détecter Vue.js. Mais ton repo est un monorepo — il faut lui dire où est le front. Dans Root Directory → mets frontend.
  • 3
    Framework Preset : Vite. Build Command : npm run build. Output Directory : dist.
  • 4
    Ajoute les variables d'environnement (VITE_API_URL, VITE_STRIPE_PUBLIC_KEY, VITE_META_PIXEL_ID) avant de cliquer Deploy.
  • 5
    Clique Deploy. Vercel va builder et te donner une URL xxx.vercel.app — ça marche comme d'habitude.
vercel.json à la racine de /frontend
// frontend/vercel.json — gère le routing Vue Router { "rewrites": [ { "source": "/(.*)", "destination": "/index.html" } ] }
⚠️
Sans le vercel.json, toutes les routes Vue Router (ex: /produit) renvoient une 404 en prod car Vercel cherche un fichier statique. Ce fichier dit à Vercel de tout rediriger vers index.html.
Domaine custom — Étape clé
Associer clipbag.fr à Vercel
C'est la partie que tu n'as jamais faite. C'est en réalité très simple — 3 DNS records à ajouter chez ton registrar (OVH, Namecheap, Gandi...).
▲
Vercel Dashboard
Ajouter le domaine
  • 1
    Dans ton projet Vercel → Settings → Domains → Add Domain.
  • 2
    Tape clipbag.fr. Vercel va te donner des DNS records à configurer. Note-les.
  • 3
    Ajoute aussi www.clipbag.fr et configure une redirection www → apex (Vercel le propose automatiquement).
🌐
Chez ton registrar (OVH / Namecheap / Gandi)
Records DNS à ajouter
TypeNom / HostValeur
A@ (ou vide)76.76.21.21 (IP Vercel)
CNAMEwwwcname.vercel-dns.com
💡
Vercel te donne exactement ces valeurs dans son dashboard. Si ton registrar ne supporte pas les enregistrements A (rare), utilise le CNAME fourni par Vercel à la place.
Propagation DNS — patience requise
  • ⏱
    La propagation DNS prend entre 5 minutes et 48h selon les registrars. OVH est généralement rapide (15-30 min).
  • ✓
    Vérifie la propagation sur whatsmydns.net — tape clipbag.fr et vérifie que le record A pointe vers 76.76.21.21.
  • 🔒
    Vercel génère le certificat SSL automatiquement (HTTPS) une fois la propagation terminée. Rien à faire de ton côté.

Stripe
Passer Stripe en production
En dev tu utilises des clés test (sk_test_). Pour vendre pour de vrai, il faut activer le compte Stripe et basculer sur les clés live.
💳
dashboard.stripe.com
Activer le compte Stripe
  • 1
    Dans Stripe Dashboard → Activate your account. Remplis les informations business (SIRET, RIB pour les virements, adresse).
  • 2
    Une fois activé → Developers → API Keys. Récupère la Publishable key (pk_live_...) et la Secret key (sk_live_...).
  • 3
    Mets sk_live_... dans la variable STRIPE_SECRET_KEY de Railway. Mets pk_live_... dans VITE_STRIPE_PUBLIC_KEY de Vercel.
🔔
Stripe → Webhooks
Configurer le webhook vers ton NestJS
  • 1
    Stripe Dashboard → Developers → Webhooks → Add Endpoint.
  • 2
    URL : https://api.clipbag.fr/stripe/webhook (adapte selon ta route NestJS).
  • 3
    Événements à écouter : payment_intent.succeeded, checkout.session.completed, payment_intent.payment_failed.
  • 4
    Récupère le Signing Secret (whsec_...) et mets-le dans STRIPE_WEBHOOK_SECRET sur Railway.
// backend — vérification webhook NestJS @Post('webhook') async handleWebhook( @Headers('stripe-signature') sig: string, @Req() req: RawBodyRequest ) { const event = stripe.webhooks.constructEvent( req.rawBody, sig, process.env.STRIPE_WEBHOOK_SECRET ); // traiter l'événement... }
Trackers & Analytics
Pixel Meta + trackers
Avec le domaine custom actif, le Pixel Meta va pouvoir être associé à clipbag.fr — ce qui améliore la qualité des données pour Meta Ads.
📊
Vue.js — main.ts ou App.vue
Intégrer le Pixel Meta
// frontend/index.html — dans <head> <script> !function(f,b,e,v,n,t,s){ if(f.fbq)return; n=f.fbq=function(){ n.callMethod ? n.callMethod.apply(n,arguments) : n.queue.push(arguments) }; if(!f._fbq)f._fbq=n; n.push=n; n.loaded=!0; n.version='2.0'; n.queue=[]; t=b.createElement(e); t.async=!0; t.src=v; s=b.getElementsByTagName(e)[0]; s.parentNode.insertBefore(t,s) }(window,document,'script', 'https://connect.facebook.net/en_US/fbevents.js'); fbq('init', 'TON_PIXEL_ID'); fbq('track', 'PageView'); </script>
Events à déclencher dans Vue.js
// Page produit — ViewContent onMounted(() => { window.fbq('track', 'ViewContent', { content_name: 'ClipBag', value: 29.99, currency: 'EUR' }); }); // Bouton panier — AddToCart function addToCart() { window.fbq('track', 'AddToCart', { value: 29.99, currency: 'EUR' }); } // Page confirmation — Purchase onMounted(() => { window.fbq('track', 'Purchase', { value: 29.99, currency: 'EUR' }); });
Vérification avec Meta Pixel Helper
  • 1
    Installe l'extension Chrome Meta Pixel Helper.
  • 2
    Navigue sur clipbag.fr, puis la page produit, puis fais une commande test. Tu dois voir PageView → ViewContent → AddToCart → InitiateCheckout → Purchase s'allumer en vert.
  • 3
    Dans Meta Ads Manager → Événements Manager → vérifie que les événements remontent bien dans l'interface.
